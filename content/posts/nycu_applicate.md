## 複試

上了丙組跟戊組，複試是一整天，早上性向測驗，下午面試。(據說)性向測驗不算分，但面試會根據寫的答案跟備審資料問問題。

### 性向測驗

丙組：很多跟考古題一樣，可以看其他人網站文章都有，但也有多幾題不一樣的

- Buffer Overflow 是什麼
- Ip Spoofing 是什麼，Android 可以做 IP Spoofing 嗎
- 怎麼 Debug kernel driver，如果不重新編譯要怎麼輸入不同參數
- 買了商用的 WiFi 基地台如果要用 C 語言加功能可以怎麼做
- TCP SYN 沒收到，要怎麼判斷機器是關機還是沒開 Port
- 設計一個 DevSecOps 的策略，要達成的目標(例如：CIA)
- 設計一個 AI Agent 評分備審資料
- 設計一個產品分類與庫存管理系統

後面兩題沒寫完。

### 面試

丙組：總共八分鐘，三位教授

自我介紹 1 分鐘，後面都抓著備審資料的內容問，有印象的：

- 你做的那些 App 是怎麼做的？為什麼用那個語言？
- 你在做校務系統主要是擔任什麼角色？不會有個資問題嗎？
- Google 黑客松做了什麼？負責什麼？

戊組：總共六分鐘，四位教授

自我介紹，也是抓備審資料問

- 你專題是做什麼？怎麼做的？

- 什麼是 memory leak？

## 建議(包含其他學校的推甄)

以下不是標準答案，參考就好。

### 比重

我認為：**成績一樣最重要，而經歷是決定「面試問得多深、教授記不記得你」。**

我報名的系所大部分是看實務能力，但還是**請以成績為基礎，請勿模仿我**。

- 推薦信
  - 如果不是要推國外學校的話，找跟你比較熟的教授就好，不一定要找有名的，但也記得提早跟教授聊，而且教授通常會要簡歷、成績單來更好了解你，請務必最先將資料整理好。
- 申請動機
  - 除了像是中正大學有特別寫到非常看重，不然我是全都交給 AI 生。
- 自傳
  - 把經歷丟給 AI 生，然後自己再修飾。
- 美術排版
  - 依據人類閱讀順序習慣，最重要的東西放前面並弄得醒目一點
- 成績
  - 我的成績不是最亮眼的，所以沒放在簡歷上，根據報名規定把成績單放在該放的位置就好。

### 專題 / 經歷領域怎麼選

在看簡章時我觀察到有些系所有加開 AI/資安組，就很適合實務能力強但成績不算頂尖的同學。

- AI
  - 應該是目前最熱門的領域，所以我認為如果要走這塊，最好要有非常突出的成果，像是發表論文、參加國際競賽、開源專案等等。
  - 缺點是太多人擠，光「會 call API、做過 side project」已經沒有鑑別度，教授看到會麻木。要有理論深度（看得懂論文、能 reproduce），或者有亮眼名次或實際落地的產品。
- 資安
  - 很缺人，但最近有跟著 AI 一起熱門起來，很卷、領域最廣，吃技術。非常非常非常建議一定要有 AIS3 跟國內外大型 CTF 經歷，有最佳專題更好。
  - 資安組通常人數少、教授記得住你，AIS3、CTF、漏洞復現、CVE、實習都記得要放。
- 網路通訊
  - 偏理論、很吃數學，比如 5G 技術、基地台之類的。
  - 如果你數學很好、喜歡推公式，這塊競爭其實比 AI/資安緩和，我也幾乎沒有認識的人在研究，但一樣可能要有論文研討會發表之類的。
- 軟體工程
  - 有在外面接過專案、實習、熟悉物件導向，可以考慮，交大丙組、中央資工軟工組都很適合。
- 量子
  - 很明顯吃數學、物理、理論，也是偏向研究型為主的領域。

> 選領域的核心原則：**挑一個能持續講半小時、而且越問越興奮的方向**，而不是挑最熱門的。

### 備審 / 簡歷怎麼寫

備審不是把做過的事全部倒出來，而是幫教授快速建立印象。

- **一頁式簡歷 + 連結**比厚厚一疊/五顏六色的樣式更有效。教授一份備審看不到幾分鐘，重點要在前半頁就跳出來。

### 面試怎麼應對

面試感覺的時間很短，幾乎都是抓備審資料問，所以：

- **自我介紹練到 1 分鐘內、不看稿**，把最強的 1～2 個亮點塞進去，引導教授往你準備好的方向問。
- 對備審裡寫的每一句話都要能展開講。
- 保持穩定的心情。
- 服裝不要太邋遢就好，我那天的服裝是牛仔外套+白色金盾獎 T+深色長運動褲+布鞋，少數不穿正裝的。

### 心態與時程

- 專題分工表、簡歷、Source Code 的整理都很花時間，不要剩不到一個月才在生。
- **當個海投戰士**，分散風險，不要省報名費，不同系所/組別看重的東西不一樣。
